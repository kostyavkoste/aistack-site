Что произойдёт в финале (выберите ОС):
1. Скачаете bundle.zip — на финальном шаге будет кнопка «📦 aistack-bundle.zip».
2. Распакуете в любую папку — например в ~/Downloads/aistack/.
3. Запустите setup-скрипт:
# Откройте Terminal (⌘+пробел → terminal):
cd ~/Downloads/aistack-bundle
bash setup.sh
# Самый простой способ — двойной клик по setup.bat в Finder:
📁 Двойной клик → setup.bat
# Или через PowerShell (правая кнопка по setup.ps1 → Run with PowerShell):
powershell -ExecutionPolicy Bypass -File .\setup.ps1
# Альтернатива — ручной маршрут через npm (PowerShell от администратора):
# 1. Node.js 22 LTS — скачать с nodejs.org, перезапустить PowerShell. Проверить:
node -v ; npm -v
# 2. Установить OpenClaw:
npm install -g openclaw@latest
# Если ошибка «cannot be loaded» — используйте npm.cmd:
npm.cmd install -g openclaw@latest
# 3. Настройка:
openclaw configure ; openclaw gateway install ; openclaw gateway restart
⚠ Если PowerShell блокирует скрипты — один раз выполните:
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
⚠ Hermes на Windows работает через тот же npm-маршрут. Порт 7777 должен быть свободен.
Проверить: Get-NetTCPConnection -LocalPort 7777 — если что-то есть, убить через Stop-Process -Id <PID> -Force
# Подключитесь к VPS по SSH, перенесите bundle (scp) и запустите:
cd ~/aistack-bundle
bash setup.sh
Рекомендация: Ubuntu 22.04 LTS. Для 24/7 работы — сервер должен быть постоянно включён.
4. Скрипт сам пошагово сделает:
  • ✓ Проверит Node.js / Git / Chrome — поставит чего нет
  • ✓ Установит OpenClaw
  • ✓ Установит Hermes (память)
  • ✓ Подключит нейросеть
  • ✓ Создаст workspace для каждого агента
  • ✓ Подключит Telegram-ботов
  • ✓ Запустит Gateway → откроется http://localhost:18789
🔄 Уже установлено? Обновить агентов без потери памяти
Если AIStack уже стоит и нужны только свежие шаблоны/скиллы/характер агентов — запустите команду ниже. Ваши MEMORY.md и USER.md не трогаются, данные сохраняются.
openclaw agents refresh-templates
Команда обновляет: workspace-templates, skills, IDENTITY.md, SOUL.md. Не трогает: MEMORY.md, USER.md, ваши проекты в ~/.openclaw/workspace-*/projects/.
⚠ важно для Windows · защита от типовых ошибок
① Используйте setup.bat двойным кликом. PowerShell не понимает синтаксис bash <(curl ...) — это даёт «Оператор "<" зарезервирован для использования в будущем». Установщик подготовит .bat-файл, который сам обойдёт ExecutionPolicy.
② Node.js ставим через nvm-windows, а не глобально. Иначе ловите npm EACCES /usr/local/lib/node_modules при установке openclaw (массовая проблема в чате клиентов).
③ setup.bat в bundle уже учитывает обе эти штуки — просто двойной клик.
⚠ важно для macOS · защита от типовых ошибок
① Если у вас стоит глобальный Node.js через brew install node — установщик попросит переустановить через nvm. Иначе при npm install -g openclaw получите EACCES permission denied в /usr/local/lib/node_modules.
② setup.sh всё проверит автоматически перед началом установки — не торопитесь.
⚠ важно для VPS · защита от типовых ошибок
① Установка плагинов на VPS занимает 2-5 минут на медленном диске (acpx, runway, tts-local-cli качают npm-зависимости последовательно). Не путайте с зависанием.
② Используйте Node.js через nvm, не глобально (root). Меньше проблем с правами на /usr/local/lib/node_modules.
③ Если уже стоит n8n / docker — порты 18789 (gateway) и 7777 (hermes) должны быть свободны.
🔴 защита от регрессии openclaw 2026.04.25
Версия openclaw 2026.04.25 сломала Gateway у десятков клиентов (закрытие connection 1006). setup-скрипт принудительно ставит стабильную openclaw@2026.4.24. Вам делать ничего не нужно — встроено.

Шаг 2 · Память команды (Hermes)

Hermes — это «память» вашей команды. Без него агенты забывают контекст между разговорами. С ним — Агент.exe помнит ваших клиентов, проекты, прошлые решения неделями и месяцами.

🧠
Что это значит на практике
Без Hermes: каждое утро объясняете команде «у меня клиент Иван, ниша салон красоты, запуск 1 июня».
С Hermes: команда сама помнит. Спросили «как там Иван?» — отвечают без напоминаний.
✓ Да REC
setup-скрипт сам поставит и подключит
